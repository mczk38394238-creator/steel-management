発注明細画面（資材部管理画面）の改修

現在 public/index.html の「資材部管理画面」（page-zaizai）に、以下4つの変更を加えてください。
order_items テーブルへの pending_weight_kg 列の追加はSupabase側で実施済みです。


変更1：暫定納期を「自由入力テキスト」にする

現在、明細追加・編集モーダルの「暫定納期」が <input type="date"> になっていますが、
実際の入力値は「4/末～」「5/中旬」のような日付ではない文字列です。
<input type="date"> を <input type="text"> に変更してください。
また、一覧表示時に tentative_delivery.slice(0,10) のように日付として加工している箇所があれば、
そのまま文字列を表示するだけに変更してください（加工不要）。


変更2：「未入荷重量(Kg)」列の追加

一覧テーブルに、「未入荷本数」の次・「備考」の前に「未入荷重量(Kg)」の列を追加してください。
データのフィールド名は pending_weight_kg です（DB側に追加済み）。
明細追加・編集モーダルにも、同じ位置に入力欄を追加してください。

テーブルの列順は最終的に以下になります：
通しNo｜商社｜メーカー｜暫定納期｜工区部位｜契約No｜鋼種｜サイズ｜規格｜長さ｜員数｜重量｜向け先｜入荷本数｜未入荷本数｜未入荷重量(Kg)｜備考｜操作


変更3：Excelから貼り付けて一括登録する機能

「資材部管理画面」の「＋明細追加」ボタンの隣に、新しく「Excelから貼り付けて登録」ボタンを追加してください。

ボタンを押したときの動作


大きめの <textarea> を1つ持つモーダルを開く（「貼り付けエリア」とラベルをつける）
ユーザーは、お手元のExcelの「ロール材」シートで、通しNo列から備考列までのヘッダー行を含まないデータ範囲（21列分）を選択してコピーし、このテキストエリアに貼り付ける
「内容を確認する」ボタンを押すと、貼り付けたテキストを解析し、プレビュー用の表を画面に表示する
プレビュー内容を確認した上で「この内容で登録する」ボタンを押すと、サーバーに送信して登録する
登録が終わったら、モーダルを閉じて一覧を再読込し、「○件登録しました」のように結果を表示する


貼り付けテキストの解析ルール（重要）

貼り付けられたテキストは、改行で行に分割し、各行をタブ文字で列に分割してください。
1行あたり21個の値が想定されますが、末尾の空セルはコピー時に省略されることがあるため、
21個に満たない場合は空文字で埋めて構いません（エラーにしない）。

21個の列は、Excelの実際の並び順そのままで、以下の対応関係になります。
2番目・5番目・8番目・9番目・17番目の列は使わない列なので、読み取った値は捨ててください。

実際の21列の並びと対応は次の通りです（左から順番）：


通しNo → tsushi_no
商社 → trading_company
メーカー → maker
暫定納期 → tentative_delivery（文字列のまま）
マスター工程使用時期 → 使わない（捨てる）
工区・部位 → section
契約No → contract_no
証明書番号 → 使わない（捨てる）
製鋼番号 → 使わない（捨てる）
鋼種 → steel_type
サイズ → size
規格 → standard
長さ(mm) → length_mm
員数 → qty
重量(Kg) → weight_kg
向け先 → destination
荷揃員数 → 使わない（捨てる）
入荷本数 → arrived_qty
未入荷本数 → pending_qty
未入荷重量(Kg) → pending_weight_kg
備考 → note


プレビュー表には、上記の「使う」列（17項目）だけを表示してください。

登録時のAPI呼び出し

既存の POST /api/order-items/bulk をそのまま使ってください。
project_id は現在選択中の物件ID、items は解析した行の配列です。
mode は指定しないでください（指定すると既存データが消える「置き換え」動作になるため、今回は必ず追記＝何回貼り付けても既存データが消えない動作にしてください）。

この機能は、1つの物件に対して何回でも繰り返し使える必要があります
（例：柱のデータを貼り付けて登録 → 後で大梁のデータを貼り付けて登録 → 後で小梁・間柱のデータを貼り付けて登録）。


変更4：一覧表のセルをクリックして直接編集できる機能

一覧テーブルの各行について、「操作」列以外のセル（通しNo～備考までの各データ）をクリックすると、
その場で入力欄に変わって直接書き換えられるようにしてください。

動作仕様


セルをクリック → そのセルの内容が <input> に置き換わり、現在の値が入った状態で編集できる
入力欄からフォーカスが外れる（他の場所をクリックする）、またはEnterキーを押すと、
そのセル1つの値だけを PUT /api/order-items/:id で更新する（他のフィールドは変更しない）
Escapeキーを押すと、編集前の値に戻して編集をキャンセルする
更新が成功したら、セルを通常の表示に戻す。失敗したらエラーを表示し、元の値に戻す


既存の「編集」ボタン（モーダルでまとめて編集する機能）はそのまま残してください。
今回追加するのは、1つのセルだけをサッと直したいときのための、もう一つの編集方法です。
